The Project: Mini-Judge

Every programming problem has three parts that can each go wrong on their own: the code that solves it, the tests that check it, and the specification that says what "correct" even means. This week you'll build all three yourself, in Rust — and on Day 3 you'll go one step further than any of the other tools can: getting an actual computer-checked proof, using a real research tool called Verus.

What is Mini-Judge?

Mini-Judge is a small tool you'll build in Rust. You give it:

  • a programming problem (we'll use "is this year a leap year?"),
  • a table of test cases (input → expected answer),
  • and any candidate function — yours, a buggy one, or one an AI wrote.

Mini-Judge runs the candidate against every test case and reports exactly which ones pass and which ones fail. That's the same idea behind real automatic grading systems, and behind VeriContest, which is what inspired this internship.

But testing only ever tells you about the inputs you actually tried. On Day 3, you'll write a formal specification and use Verus — a real verification tool for Rust, the same kind of tool used in the research behind VeriContest — to get a mathematical guarantee that your code is correct for every possible input, not just the ones in your test table.

On Thursday, you'll point all of this at code written by an AI, and use it to answer a real research question: does giving the AI a precise, formal specification (instead of a vague description) make its code more likely to actually verify? You'll have the data to answer that, because you'll have built the tool that measures it.

How the week builds up

The project is split into stages we call tiers. Each tier is a complete, working version of the project — so even if the week goes slower than planned, whatever tier you've reached is still a finished thing you built. Why tiers instead of one big spec? Finishing Tier 1 alone is a complete, real project. Everything after that makes it better, but nothing after Tier 1 is required for the week to have been a success. If you ever finish faster than expected, the Extensions page always has a next step ready — you'll never run out of things to build.
A heads-up: Rust and Verus have more to learn upfront than a language like Python would. That's expected, and it's exactly why the tiers exist — every code example on this site is given to you working and ready to copy into the playground, so you're reading and adapting real code, not writing it from a blank page.
