Glossary

Test case

A single (input, expected output) pair used to check a solution.

Edge case

An unusual input (a boundary, an empty value, a huge number) where bugs like to hide.

Specification

A precise rule that defines the correct answer for every possible input, not just examples.

spec fn

A Verus function that exists only to describe a specification mathematically: it's never actually run.

exec fn

An ordinary Rust function that actually runs. Most functions you write are exec functions.

proof fn

A Verus function containing a logical argument (a proof) rather than a running program, used for facts that need more help than a simple ensures clause.

requires / ensures

A precondition (what must be true going in) and a postcondition (what's guaranteed coming out) on a function.

SMT solver

A program (Verus uses one called Z3) that automatically checks whether a logical statement is always true, always false, or neither: the engine that actually does Verus's proving.

Verified vs. tested

Tested = checked on the inputs you tried. Verified = proven correct for every possible input, by an SMT solver rather than a human.

pass@k

The fraction of k independent AI attempts that fully succeed: a measure of consistency, not just luck.

Go further

  • Verus: the tool you used on Day 4. The official tutorial (verus-lang.github.io/verus/guide) covers everything from this week's basics up through loops, data structures, and much harder proofs.
  • Dafny: a similar but older and more teaching-oriented verification language, often used as a gentler first introduction to the same ideas.
  • Property-based testing: libraries like proptest (Rust) or Hypothesis (Python) automatically generate random test inputs and search for the smallest failing example, sitting a step below full verification.
  • Code-generation benchmarks: research papers like HumanEval and MBPP measure how often AI models write fully-correct code on curated problem sets, using the pass@k idea from the Extensions page.
  • VeriContest: the research effort behind this internship; ask Zichen or Wenxi for the full picture and current results.
