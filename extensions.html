E1 · A second problem: password validator (plain Rust)

Reuse your judge exactly as-is — only the problem and the candidate functions change. Rules: at least 8 characters, at least one digit, at least one uppercase letter. Design a test table and find the bugs in the provided buggy versions.

This extension deliberately stops at testing, not a Verus proof. Proving facts about strings in Verus requires reasoning about sequences of characters (Verus's Seq type and quantifiers over indices), which is genuinely more advanced than the integer arithmetic in the leap-year proof. Ask a mentor if you want to attempt it as a stretch-stretch goal — it's a great "what's next" question, just not a quick add-on.

E2 · Mutation round: "who tests the tests?"

A test suite is only as good as the bugs it catches. Here's how to check yours:

  1. Take a correct function (yours, or the reference one) and secretly plant 2 bugs in a copy of it — small changes, like swapping && for ||, or an off-by-one in a comparison.
  2. Swap your mutated code with your partner, without telling them what you changed.
  3. Run their test suite against your mutant. Score: bugs caught out of bugs planted.
  4. Whoever's tests catch more bugs (relative to how sneaky the bugs were!) wins bragging rights. Discuss: what kind of test case would have caught the bug that got missed?
